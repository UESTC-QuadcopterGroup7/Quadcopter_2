# STM32F401RE 内存架构详解

本文针对本项目使用的 **STM32F401RE** 芯片，详细剖析其内部 Flash 和 SRAM 的物理构成、地址映射及特性。相比于 F1 系列，F4 系列在 Flash 扇区结构和总线架构上有显著差异。

---

## 1. 内存概览 (Memory Map)

STM32F401RE 基于 Cortex-M4 内核，采用哈佛架构（指令与数据总线分离），但在物理上统一编址在 4GB 空间内。

| 区域 | 起始地址 | 大小 (Size) | 用途 |
| :--- | :--- | :--- | :--- |
| **Code (Flash)** | `0x0800 0000` | 512 KB | 存放用户程序代码、常量 |
| **SRAM** | `0x2000 0000` | 96 KB | 存放变量、堆栈 |
| **System Memory** | `0x1FFF 0000` | 30 KB | 出厂固化的 Bootloader |
| **OTP** | `0x1FFF 7800` | 512 Bytes | 一次性可编程区域 (用户数据) |
| **Option Bytes** | `0x1FFF C000` | 16 Bytes | 读写保护、看门狗配置等 |

---

## 2. Flash 存储器详解

STM32F4 的 Flash 并不是像 SRAM 那样均匀分布的，它由**不同大小的扇区 (Sectors)** 组成。这在进行 Flash 擦写操作（如保存参数）时非常重要，因为**Flash 只能按扇区擦除**。

### 2.1 扇区布局 (Sector Layout)
STM32F401RE (512KB) 的 Flash 分为 8 个扇区：

| 扇区编号 | 扇区大小 | 地址范围 | 建议用途 |
| :--- | :--- | :--- | :--- |
| **Sector 0** | 16 KB | `0x0800 0000` - `0x0800 3FFF` | **ISR 向量表 + 启动代码** |
| **Sector 1** | 16 KB | `0x0800 4000` - `0x0800 7FFF` | **关键配置参数** (小容量适合擦写) |
| **Sector 2** | 16 KB | `0x0800 8000` - `0x0800 BFFF` | **参数存储 / 日志** |
| **Sector 3** | 16 KB | `0x0800 C000` - `0x0800 FFFF` | **参数存储 / 日志** |
| **Sector 4** | 64 KB | `0x0800 0000` - `0x0801 FFFF` | **主程序代码** |
| **Sector 5** | 128 KB | `0x0802 0000` - `0x0803 FFFF` | **主程序代码 / 固件升级区** |
| **Sector 6** | 128 KB | `0x0804 0000` - `0x0805 FFFF` | **主程序代码 / 图片资源** |
| **Sector 7** | 128 KB | `0x0806 0000` - `0x0807 FFFF` | **主程序代码 / 文件系统** |

**设计启示**：
1.  **参数保存**：尽量使用 Sector 1~3 (16KB) 来保存用户配置参数。因为擦除一个 128KB 的扇区（Sector 5-7）比擦除 16KB 扇区要慢得多，且会暂停 CPU 执行更久。
2.  **代码位置**：编译器通常从 Sector 0 开始填充代码。

### 2.2 Flash 等待周期 (Wait States)
由于 Flash 的读取速度（约 24MHz）远低于 CPU 主频（84MHz），CPU 直接读 Flash 会跟不上。
STM32F4 引入了 **ART Accelerator (自适应实时加速器)** 和 **预取缓冲 (Prefetch Buffer)**。
*   在 84MHz 下，通常需要设置 **2~3 个等待周期 (Latency)**。
*   开启 ART 加速后，CPU 从 Flash 取指可以达到接近 0 等待的性能。

## 3. SRAM 存储器详解

STM32F401RE 拥有 **96 KB** 的片内 SRAM，通常称为 SRAM1。
不同于 F407 等高级型号（有 SRAM1, SRAM2, CCM），F401 的 SRAM 是连续的一块。

*   **起始地址**：`0x2000 0000`
*   **结束地址**：`0x2001 7FFF`
*   **特性**：
    *   **零等待**：CPU 读写 SRAM 不需要等待周期，速度极快。
    *   **掉电丢失**：复位或断电后数据消失。
    *   **用途**：
        1.  **全局变量 (.data, .bss)**：静态分配的内存。
        2.  **堆 (Heap)**：`malloc` 动态分配的内存。
        3.  **栈 (Stack)**：函数调用、局部变量、中断现场保护。

### 3.1 什么是 CCM RAM？ (本型号没有)
**注意**：STM32F407/429 等高端型号有 **CCM (Core Coupled Memory)**，直接挂在内核 D-Bus 上，速度最快但不能被 DMA 访问。
**STM32F401RE 没有 CCM RAM**，它的 96KB 都可以被 CPU 和 DMA（如 ADC DMA 传输、串口 DMA 发送）正常访问，无需担心 DMA 访问权限问题。

---

## 4. 系统存储器 (System Memory) 与 Bootloader

地址：`0x1FFF 0000`
这是一块 **ROM (Read-Only Memory)**，在芯片出厂时由 ST 固化了一段代码（Bootloader）。

**作用**：
当 `BOOT0` 引脚拉高，`BOOT1` 拉低（或通过 Option Bytes 配置）复位时，CPU 会从这里启动。
这段代码支持通过 **USART1 (PA9/PA10)**、**USB (PA11/PA12)** 或 **I2C** 等接口，将用户的程序下载到 Flash 中（即 ISP 下载）。

---

## 5. OTP 区域 (One-Time Programmable)

地址：`0x1FFF 7800` (512 字节)
这是一块特殊的 Flash 区域，只能从 1 写为 0，不能擦除（不能变回 1）。

**用途**：
*   产品序列号
*   软件加密密钥
*   硬件版本标识

---

## 6. 总结与建议

1.  **Flash 分区策略**：如果你要做“断电保存参数”的功能，请务必避开 Sector 0（放了代码），推荐使用 **Sector 1 (0x0800 4000)**，因为它是 16KB 小扇区，擦除快，且通常代码量不会大到覆盖它（除非代码超过 16KB）。
2.  **RAM 使用**：F401 的 96KB 相对充裕。如果遇到栈溢出，检查是否定义了过大的局部数组（如 `char buffer[10240]`），建议改用全局变量或静态变量以移到 .bss 段。
3.  **DMA 注意**：在 F401 上，所有 SRAM 均可用于 DMA，这点比 F407 方便。
